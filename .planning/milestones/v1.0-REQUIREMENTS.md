# Requirements: Minance UX Polish

**Defined:** 2026-04-02
**Core Value:** A polished, reliable personal finance tracker you can trust with your financial data — clean interactions, clear feedback, and intuitive layouts.

## v1 Requirements

### Filter UX

- [x] **FILT-01**: Single unified search input replacing duplicate query/merchant inputs with fuzzy matching across merchant, description, notes, and tags
- [x] **FILT-02**: Active filter badges displayed outside filter panels with remove capability for each active filter
- [x] **FILT-03**: URL state synchronization for all filters enabling shareable/bookmarkable filtered views
- [x] **FILT-04**: Debounced search input (300ms delay) to reduce unnecessary API calls
- [x] **FILT-05**: Date range presets (This Month, Last 30 Days, Last 90 Days, This Year, Custom)
- [x] **FILT-06**: Multi-select dropdowns for categories filter (replacing single-select)
- [x] **FILT-07**: Multi-select dropdowns for accounts filter (replacing single-select)
- [x] **FILT-08**: Clear/reset all filters functionality

### Visual/Layout

- [x] **VISL-01**: Consistent table styling with right-aligned amounts and color semantics (green/income, red/expense)
- [x] **VISL-02**: Loading states during data operations with appropriate spinners/skeletons
- [x] **VISL-03**: Empty state messaging when no transactions match filters
- [x] **VISL-04**: Consistent spacing and visual hierarchy across all pages
- [x] **VISL-05**: Sticky table headers for improved scannability when scrolling

### Interaction Patterns

- [x] **INTX-01**: Inline form validation on blur with specific, actionable error messages
- [x] **INTX-02**: Input preservation on validation error (users don't lose entered data)
- [x] **INTX-03**: Toast notifications for confirmations (success, error, info) with 3-5 second auto-dismiss
- [x] **INTX-04**: Optimistic updates for common actions with rollback on error
- [x] **INTX-05**: Undo functionality for destructive actions (delete, bulk delete) via toast with undo button
- [x] **INTX-06**: Command palette (cmd+k) for quick navigation and common actions

### User Feedback

- [x] **UFBK-01**: Clear error messages with recovery actions (not generic "Something went wrong")
- [x] **UFBK-02**: Success confirmations for important actions (save, delete, import)
- [x] **UFBK-03**: Loading indicators during async operations (buttons, tables, forms)
- [x] **UFBK-04**: Disabled states for unavailable actions with visual indication

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Advanced Features

- **FILT-09**: Smart search with natural language queries ("groceries last month")
- **FILT-10**: Saved filter presets for recurring views
- **VISL-06**: Bulk action support for multi-row operations
- **INTX-07**: Keyboard shortcuts beyond command palette
- **INTX-08**: Progressive disclosure for advanced filter options

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Mobile app | Web-first, responsive design sufficient |
| Multi-user collaboration | Single-user personal finance app |
| Real-time sync | Not needed for personal use |
| Payment integrations | Tracking only, no transaction execution |
| Offline mode | Requires significant architecture changes |
| Data export formats | CSV import exists; export deferred |
